初识长连接（1）- websocket

Heartape 2023-03-06 websocket

# 1.简介

WebSocket在HTML5提出，属于比较新的协议，所以有一定的浏览器兼容问题。 WebSocket和Http一样，都是基于都TCP，属于应用层的协议。 作为全双工的通信协议，它允许服务端和客户端双向推送。

import jakarta.websocket.*;
import jakarta.websocket.server.ServerEndpoint;
import lombok.extern.slf4j.Slf4j;

/**
 * ws://localhost:8080/test
 */
@Slf4j
@ServerEndpoint("/test")
public class WebsocketServer {

    /**
     * 连接成功
     */
    @OnOpen
    public void onOpen(Session session) {
        log.info("开启连接");
    }

    /**
     * 连接关闭
     */
    @OnClose
    public void onClose() {
        log.info("关闭连接");
    }

    /**
     * 接收到消息
     */
    @OnMessage
    public void onMessage(String message, Session session) {
        log.info(message);
    }

    /**
     * 异常
     */
    @OnError
    public void onError(Throwable throwable) {
        log.info("连接异常");
    }
}

# 3.探究

# 1.原理

  • websocket与http极为相似，甚至起初的建立连接就是使用的http。
  • 随后，客户端会向服务器请求协议升级，所以需要服务器支持。
  • 最后会将该当前连接作为websocket继续利用。

# 2.分析

有以下几点需要注意:

  • 101是http状态码，而非websocket，意为协议转变。
  • 无需重复建立连接，在持续传输时资源开销较少。
  • "Sec-WebSocket-Key"加上一个特殊字符串"258EAFA5-E914-47DA-95CA-C5AB0DC85B11"，然后计算SHA-1摘要，之后进行Base64编码，将结果做为"Sec-WebSocket-Accept"头的值。
