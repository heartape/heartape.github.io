# 3.CyclicBarrier

CyclicBarrier(循环屏障)是一种同步辅助工具，允许一组线程都等待对方到达一个共同的障碍点。CyclicBarrier在涉及固定大小的线程组的程序中很有用，这些线程偶尔必须相互等待。屏障被称为循环的，因为它可以在等待线程释放后重新使用。CyclicBarrier支持一个可选的Runnable命令，该命令在参与方中的最后一个线程到达之后，但在释放任何线程之前，每个屏障点运行一次。此屏障操作有助于在任何一方继续之前更新共享状态。

CyclicBarrier最大的使用场景便是线程间数据同步，每一批等待的线程都有极大的可能获取相同的数据

public class CyclicBarrierCode {
    public static Config config = null;

    public static CyclicBarrier cyclicBarrier = new CyclicBarrier(3, () -> {
        System.out.println("sync start");
        config = new Config();
        System.out.println("sync end");
    });
}

还有一种用途便是和并请求,在回调中进行合并

public class CyclicBarrierCode {
    public final BlockingQueue<Order> queue = new ArrayBlockingQueue<>(2);
    public final CyclicBarrier cyclicBarrier = new CyclicBarrier(2, () -> {
        Order order1 = queue.poll();
        Order order2 = queue.poll();
        int count = order1.count + order2.count;
        order1.status = 1;
        order2.status = 1;
        System.out.println("merge count :" + order1.count + "+" + order2.count + "=" + count);
    });

    static class Order {
        int id;
        int goodsId;
        int count;
        int status;

        public Order(int id, int goodsId, int count, int status) {
            this.id = id;
            this.goodsId = goodsId;
            this.count = count;
            this.status = status;
        }
    }
}

# 4.SemaphoreCode

SemaphoreCode计数信号灯。从概念上讲，信号量维护一组许可。如果有必要，每次收购都会阻止，直到获得许可证，然后再获得许可证。每次发行都会添加一个许可证，可能会释放一个阻止收购的人。然而，没有使用实际的许可对象；信号量只是保持对可用数字的计数并相应地进行操作。

总的来说，SemaphoreCode就是用来限制线程并发。

比如，可以严格限制线程的并发数。只是需要保证release()执行，或者设定补偿机制，防止许可证耗尽导致死锁。

public class SemaphoreCode {
    private final Semaphore semaphore = new Semaphore(2, true);
    
    public void doTask(){
        try {
            semaphore.acquire();
            TimeUnit.SECONDS.sleep(3);
            System.out.println("do task");
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        } finally {
            semaphore.release();
        }
    }
}

再者，可以实现一个简单的令牌桶算法。

public class SemaphoreCode {
    private final Semaphore semaphore = new Semaphore(5, true);
    
    private void init(){
        for (;;) {
            semaphore.release();
            try {
                TimeUnit.SECONDS.sleep(1);
            } catch (InterruptedException ignore) {}
        }
    }
    
    public void doTask(){
        try {
            semaphore.acquire();
            System.out.println("finish");
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
}

# 5.CountDownLatch

CountDownLatch是一种同步辅助工具，允许一个或多个线程等待，直到在其他线程中执行的一组操作完成。

  • 用门闩来形容很形象。
  • 类似于局部和整体的关系，整体等待局部完成。
  • 可以理解为对任务优先级的区分，重要的任务会优先执行。
import java.util.concurrent.CountDownLatch;

public class CountDownLatchCode {
    private CountDownLatch countDownLatch = new CountDownLatch(3);

    public void resetPriority(){
        countDownLatch = new CountDownLatch(3);
    }
    
    public void highPriority(){
        try {
            TimeUnit.SECONDS.sleep(2);
            countDownLatch.countDown();
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }

    public void lowPriority(){
        try {
            countDownLatch.await();
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
}
