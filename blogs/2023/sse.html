初识长连接（2）- sse

Heartape 2023-03-08 sse

# 1.简介

sse即Server-Sent Events是一项服务器主动推送技术，仅可以单工通信。 sse是一个及其容易被忽视的一项技术，我也是最近才了解到。 在websocket的光辉之下，sse显得没那么强大，但是良好的兼容性和轻量化的协议让它在仅需服务器推送的场景下更为合适。

# 2.代码

@RestController
@RequestMapping("/sse")
public class SseService {
    /**
     * 通过将SseEmitter缓存，可以后续持续推送
     * @return SseEmitter,返回后才会生效。
     */
    @CrossOrigin
    @GetMapping("/message")
    public SseEmitter create(){
        SseEmitter sseEmitter = new SseEmitter(0L);
        sseEmitter.onCompletion(() -> System.out.println("Completion"));
        sseEmitter.onTimeout(() -> System.out.println("Timeout"));
        sseEmitter.onError(e -> System.out.println("Error"));
        threadPoolExecutor.execute(() -> {
            for (int i = 0; i < 100; i++) {
                sseEmitter.send(SseEmitter.event()
                        .id(timestamp)
                        .data("message" + i)
                );
            }
        });
        return sseEmitter;
    }
}
