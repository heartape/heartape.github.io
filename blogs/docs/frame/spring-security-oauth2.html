spring security oauth2

Heartape 2022-12-31 oauth2spring security

# 1.简介

目前oauth2应用较为广泛，配合oidc可以实现公共的认证授权的服务。 但目前提供oauth2认证服务的产品并不规范，甚至可以说是混乱。 spring官方也意识到，所以废弃了原本spring-security-oauth2，将oauth2做成了spring-security的插件。 后来迫于社区压力，又开启了spring-authorization-server项目，至此spring的oauth2算是重新建立。

注意！本文仅讨论spring的oauth2体系，与其他产品会有不同。

# 2.流程分析

一般来说，客户端、资源服务器独立运行，仅在进行认证时依赖于认证服务器。

要分析流程，就不得不从各个服务的配置说起：

  • client指向authorization-server
  • resource-server指向authorization-server（也可以将公钥配置到资源服务器）
  • authorization-server保存各个账户的认证授权信息（包括合法的认证方式和回调地址），某些情况也会保存用户个人信息

用户可感知的oauth2流程（以授权码模式为例）：

  • 用户选择一个认证中心进行登录
  • 随后进行授权
  • 跳转回首页

实际的oauth2流程（以授权码模式为例）：

  • user选择认证中心
  • 验证client账号密码,并将验证字段state发送到server
  • user登录
  • user授权
  • server返回授权码和验证字段，client对其校验
  • client获取token
  • client获取userinfo
  • 跳转client根地址
  • client从资源服务器获取数据

# 3.源码分析

# 1.client

过滤器:

  1. OAuth2AuthorizationCodeGrantFilter: 接收授权码，然后验证并存储
  2. OAuth2LoginAuthenticationFilter: server回调,返回code和state

# 2.resource-server

  1. OAuth2ResourceServerJwtConfiguration: 配置jwk-set-uri，用于请求server并获取公钥
  2. RestOperationsResourceRetriever: 通过http请求向server发送请求

# 3.authorization-server

过滤器:

  1. OAuth2AuthorizationServerMetadataEndpointFilter: 元信息
  2. OAuth2AuthorizationEndpointFilter: 认证
  3. OAuth2TokenEndpointFilter: token
  4. NimbusJwkSetEndpointFilter: 公钥

# 4.其他

  • 由于spring采用了session来存储授权码、验证字段和token等，在分布式情况下会存在请求失败的问题。目前官方建议使用spring-session来共享。
  • 可以通过自定义userMapper修改userinfo。
  • 可以配合gateway使用，由于spring-security采用的是过滤器，会优先生效。
  • rsa密钥（可以通过openssl生成）中的kid没有密码学上的意义，只是用来寻找密钥的索引。
