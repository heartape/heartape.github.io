redis深入探究(1) - RESP协议

Heartape 2022-09-22 redisRESP

# 1.简单介绍

RESP是Redis客户端与Redis服务器相互通信时使用的私有协议， 全称REdis Serialization Protocol ，即redis串行协议。 RESP 协议是一个应用层协议，也就是说它的底层还是依赖于 tcp/ip。

# 2.客户端

redis同时支持内联命令，和RESP协议，这里我们不谈内联命令。 以set命令为例

"SET key value"
==>
"*3"
"$3"
"SET"
"$3"
"key"
"$5"
"value"

  • '*'表示数组长度

# 3.服务端

服务端相对而言就复杂多了，需要考虑多种情况。我做了如下总结：

  • 单行回复（简单固定字符串回复），回复的第一个字节将是“+”
  • 多行回复（长字符串回复），回复的第一个字节将是“$”
  • 批量回复（数组回复），回复的第一个字节将是“*”
  • 错误消息，回复的第一个字节将是“-”
  • 整型回复（正整形数字回复），回复的第一个字节将是“:”
单行回复， 如：set key value  ==>  +OK, ping  ==>  +pong
多行回复， 如：get key  ==>  $5 value ,如:get key(不存在的key)  ==>  $-1 
批量回复，如：
    keys * ==>
    *3 -> 总共3条数据返回 
    $29 -> 数据字符数 
    like:ar:po:731833596062466048 -> 数据 
    $30 
    like:ar:sp:1518946330134302721 
    $7 
    heat:ar 
    典型的数组嵌套的情况：
    scan 0 match * count 100 ==>
    *2
    $1
    0 -> 下一次游标
    *46 -> 所有的key
    $3
    key
    ......(剩下的45个key)
错误消息，如:het(不存在的命令) key ==> -ERR unknown command `het`,...... 
整型回复(只有在hash等一个主key对应多个数据时才会返回),如：hset hkey key value ==>  :1
