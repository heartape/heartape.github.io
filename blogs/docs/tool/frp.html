frp内网穿透

Heartape 2022-10-31 frpnetwork

# 1.简介

突然发现idea有一个黑科技--code with me，可以实现远程协同，使用配置较高的机器写代码，而且非常流畅。 但是有一个问题，需要在其他机器连接时，分享的机器需要点击同意才可以连接。 起初找了一个图像识别的小工具，但是需要配置环境，而且限制比较大。 最终采用了远程桌面加frp内网穿透的方式。

# 2.搭建

仓库地址

linux下载

windows下载

官方文档，很详细，养成看文档的好习惯，有问题的话issuse应该都可以找到

启动命令:./frps -c ./frps_full.ini 使用frps_full.ini作为配置可能启动失败，因为有插件配置却没有安装插件，服务端配置文件中去掉插件配置即可

客户端配置

[common]
server_addr = x.x.x.x
server_port = 7000

[remote-connect]
type = tcp
local_ip = 127.0.0.1
local_port = 3389
# 不知道为什么3389不能使用
remote_port = 13389

# 3.使用 systemd

创建配置文件：vim /etc/systemd/system/frps.service

[Unit]
# 服务名称，可自定义
Description = frp server
After = network.target syslog.target
Wants = network.target

[Service]
Type = simple
# 启动frps的命令，需修改为您的frps的安装路径
ExecStart = /path/to/frps -c /path/to/frps.ini

[Install]
WantedBy = multi-user.target
