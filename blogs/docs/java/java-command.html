# 4.jps,jstack

jps用于打印当前所有的java进程 jstack用于打印当前进程所有的线程堆栈

public class Test {
    public static void main(String[] args) {

        Map<String, String> map1 = new ConcurrentHashMap<>();
        Map<String, String> map2 = new ConcurrentHashMap<>();

        Thread thread1 = new Thread(() -> {
            synchronized (map1) {
                map1.put("thread1", "执行");
                try {
                    TimeUnit.SECONDS.sleep(2);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (map2) {
                    map2.put("thread1", "执行");
                }
            }
        });
        Thread thread2 = new Thread(() -> {
            synchronized (map2) {
                map2.put("thread2", "执行");
                try {
                    TimeUnit.SECONDS.sleep(2);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (map1) {
                    map1.put("thread2", "执行");
                }
            }
        });

        thread1.start();
        thread2.start();
    }
}

jstackd打印对应进程的线程堆栈，由打印出的信息可以看出程序死锁

另外在排查问题时还有其他系统命令:

  • ps -ef|grep PID(查看启动命令)
  • ps -mp PID -o THREAD,tid,time(查看cpu占用，不推荐)
  • top -b -n 1 | grep java| awk '{print "PID:"$1",CPU percent:"$9"%",mem:"$6/1024" mb,"mem percent:"$10"%"}'(查看所有java进程硬件占用，推荐)
  • top p PID(shift + h 可以展开所有线程)(动态查看硬件占用，推荐) 拓展一个java后台启动命令:nohup java -jar xxx.jar >/dev/null 2>&1 &。
