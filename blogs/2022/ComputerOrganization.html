计算机组成原理

Heartape 2022-06-13 计算机

# 1. 计算机系统概论

# 1. 计算机系统

计算机系统包括硬件和软件;软件包括系统软件和应用软件;

系统软件:

  • 语言处理程序
  • 操作系统
  • 服务性程序
  • 数据库管理系统
  • 网络软件

程序执行的过程:

  1. 高级语言
  2. 汇编语言
  3. 操作系统
  4. 机器语言
  5. 微指令系统

# 2. 计算机基本组成

系统复杂性管理的方法:

  • 层次化
  • 模块化
  • 规则性

当前计算机采用冯.诺依曼体系,由五大部分组成:

  • 运算器
  • 控制器
  • 存储器
  • 输入设备
  • 输出设备

# 3. 计算机硬件指标

CPI:执行一条指令所需的时钟周期数

MIPS:每秒执行百万条指令

FLOPS:每秒浮点运算次数

# 2. 总线

# 1. 总线的定义

总线是连接各个部件的信息传输线,是各个部件共享的传输介质.

# 2. 总线结构

  1. 单总线:会造成总线争用;总线过长导致延迟大;

  2. 面向CPU的双总线结构:I/O总线+M总线:主存和外部设备传输数据时,CPU计算会被打断;

  3. 以存储器为中心的双总线结构:系统总线+存储总线:两条总线无法同时工作,还有一定的缺陷;

  4. 三总线1:I/O总线+主存总线+DMA总线

  5. 三总线2:局部总线+系统总线+扩展总线

  6. 四总线:局部总线+系统总线+高速总线+扩展总线

  7. PCI总线:可以多层扩展

# 3. 总线分类

# 1. 片内总线

芯片内部的总线

# 2. 系统总线

  • 数据总线:双向,和CPU交换数据
  • 地址总线:单项,存储设备寻址
  • 控制总线:有出有入,存储器读写,总线允许&中断确认

# 3.通信总线

  • 串行总线
  • 并行总线

# 4.总线特性

  1. 机械特性

  2. 电气特性:传输方向和有效电平范围

  3. 功能特性:总线传输的信号的功能:地址;数据;控制;

  4. 时间特性:信号时序关系

# 5.总线性能指标

# 1.总线宽度

数据线的根数

# 2.标准传输率

每秒传输的最大字节数

# 3.时钟同步/异步

传输方式同步or不同步

# 4.总线复用

将地址线与数据线共用,减少的管脚数,减小芯片体积

# 5.信号线数

地址线,数据线和控制线的总和

# 6.总线控制方式

突发,自动,仲裁,逻辑,计数

# 7.其他指标

负载能力

# 6.总线标准

# 7.总线控制

# 1.总线判优控制

主设备:对总线有控制权

从设备:响应从主设备发来的总线命令

方法:

  • 集中式:链式查询;计数器定时查询;独立请求方式
  • 分布式

# 2.总线通信控制

总线传输周期:

  1. 申请分配阶段:主设备申请,总线仲裁决定;
  2. 寻址阶段:主设备通过地址找到从设备,通过命令控制从设备;
  3. 传数阶段:主设备和从设备交换数据;
  4. 结束阶段:主设备撤销信息.

# 3.总线通信的四种方式

1.同步通信:由统一时标控制数据传输,因为主从设备需要同步,所以一般在两者速度差距不大且总线长度短时使用;

2.异步通信:采用应答方式,没有公共时钟标准;

3.半同步通信:同步异步结合(WAIT信号低电平代表尚未准备传输数据)

4.分离式通信:

  • 各模块均有权申请占用总线;
  • 主设备申请总线后立即放弃总线,等待从设备响应,而总线不会等待;
  • 从设备(此时相当于主设备)响应后申请总线并传输数据;
  • 除去等待时间与同步通信一致;
  • 充分利用总线;

# 3.存储器

# 1.分类

存储介质:

  • 半导体存储器:TTL,MOS
  • 磁表面存储器:磁头,磁载体
  • 磁芯存储器:硬磁材料,环状原件
  • 光盘存储器:激光,磁光材料

存储方式:

  • 存取时间与物理地址无关(随机访问):随机存储器;只读存储器;
  • 存取时间与物理地址有关(串行访问):顺序存取存储器,磁带;直接存取存储器,磁盘;

作用:

  • 主存储器:RAM;ROM
  • Flash Memory
  • 高速缓存
  • 辅助存储

结构体系:

  1. 寄存器
  2. 缓存
  3. 主存
  4. 磁盘
  5. 光盘
  6. 磁带

# 5.存储器的校验

合法代码集合:计算机中可能存在的二进制数据,出现其之外的数据表明出现了错误.

编码的最小距离:任意两组合法代码之间二进制位数的最小差异.

L-1=D+C(D>=C)

  • L----编码的最小距离
  • D----检测错误的位数
  • C----纠错的位数

汉明码

偶校验:1的个数为偶数个.

2^k>=n+k+1(k为增加的检测位的个数,n为原数据位数);

例如:
按配偶原则配置0011的汉明码
k=3
汉明码:C1,C2,0,C3,0,1,1
C1组包括位置序号为XXX1的数,比如1,3,5,7
C2组包括位置序号为XX1X的数,比如2,3,6,7
C3组包括位置序号为X1XX的数,比如4,5,6,7
C1=0,0,1=1
C2=0,1,1=0
C3=0,1,1=0
则汉明码:1,0,0,0,0,1,1

纠错
P按照C分组
P1=1,0,0,1=0正确
P2=0,0,1,1=0正确
P4=0,0,1,1=0正确
P=P4P2P1=000
P的值就是错误的项,0代表没有

# 3.高速缓冲存储器

# 1.需求

CPU与DRAM速度差异过大,需要缓存;

程序访问局部性原理:

  • 时间局部性:被使用过的数据可能还被使用;
  • 空间局部性:被使用的数据的相邻数据也可能被使用;

# 2.工作原理

将主存中整个块的数据,块号以及块内偏移量存到缓存中,并用标记表示缓存中对应在主存中地址.

# 3.命中

如果CPU需要的数据已被调入缓存中,称为缓存命中;

CPU给出的主存地址一般包括:主存字块标记,cache字块标记,字块内地址;

**读:**如果未命中,则先将主存中需要的数据传给CPU,再将数据放到缓存中;

写:

  • 写直达法(一致):既写入缓存又写入主存,缓存块退出时,不需要对主存进行操作;
  • 写回法(不一致):只写入缓存,缓存块退出时,再写入主存;

缓存命中率和缓存大小和块长有关;

块长一般为4--8个字节;

缓存--主存系统的效率e: $$ e=\frac{访问缓存的时间}{平均访问时间}*100% $$ 其中平均访问时间包括缓存访问和主存访问.

# 4.输入输出系统

# 1.I/O接口

# 1.设置接口的目的

  • 实现设备的选择
  • 实现数据缓冲达到速度匹配:设备速度不一致
  • 实现数据串---并格式转换
  • 实现电平转换
  • 传送控制命令
  • 反映设备状态:接口中有状态标志忙,正常,占用

# 2.总线连接方式的I/O接口电路

  • 设备选择线
  • 数据线:命令寄存器,命令译码器
  • 命令线:数据缓冲寄存器
  • 设备状态线

状态:

  • 完成触发器D
  • 工作触发器B
  • 中断请求触发器INTR
  • 屏蔽触发器MASK:值为1表示无法发送中断请求;

# 2.程序查询

D表示数据输入完成,等待CPU读取,CPU会一直等待D的状态;B表示设备输入完成,工作结束;

设备启动时,D=0,B=1;随后工作结束D=1,B=0;

# 3.程序中断

# 1.中断的概念

外部设备请求与CPU传输数据导致CPU暂停主程序去运行中断服务程序.

# 2.中断流程

  1. 现场保护:

    程序断点保护:中断指令完成

    寄存器内容保护:进栈指令

  2. 中断服务:不同的设备有不同的设备服务

  3. 恢复现场:出栈指令

  4. 中断返回

    中断分类:

  • 单重中断:不允许中断现行的中断服务;
  • 多重中断:允许更高级别的中断源中断现行的中断服务;

# 4.DMA

数据传送不经过CPU,而是由DMA控制器实现.

# 1.DMA与主存交换数据的方式

  • 停止CPU访问主存:未充分利用主存
  • 周期挪用(周期窃取):充分利用主存
  • DMA与CPU周期交替访问:实现简单

# 2.DMA接口的功能

  1. 向CPU申请DMA传送;
  2. 处理总线控制权的转交;
  3. 管理系统总线,控制数据传送;
  4. 确定数据传送的首地址和长度,修正传送过程中的数据地址和长度;
  5. DMA传送结束时,给出操作完成信号;

# 3.DMA接口组成和数据传输过程

传输过程:

  1. 预处理:1.通知DMA控制传送方向;2.设备地址--DAR;3.主存地址--AR;4.传送字数--WC;

  2. 数据传送;

  3. 后处理(由中断服务程序完成):1.校验数据;2.是否继续用DMA;测试传送过程是否正确,错则转诊断程序;

# 4.DMA与程序中断比较

# 7.指令系统

# 1. 指令系统基本概念

指令集：一台机器所有指令的集合。系列机（同一公司不同时期生产）；兼容机（不同公司生产）

指令字长：指令中包含的二进制位数，有等长指令、变长指令。

指令分类

    • 根据层次结构：高级、汇编、机器、微指令

    • 根据地址码字段个数：零、一、二、三地址指令

    • 根据操作数物理位置

      • 存储器-存储器（SS）
      • 寄存器-寄存器（RR）
      • 寄存器-存储器（RS）

    • 根据指令功能：传送、算术运算、位运算、控制转移

指令格式：操作码+数据源+寻址方式

# 2. 寻址方式

**指令寻址方式：**顺序寻址，跳跃寻址

操作数寻址方式：

  • 立即寻址：地址码字段是操作数本身;
  • 寄存器寻址：地址码字段是寄存器地址;
  • 直接寻址：地址码字段是内存地址;
  • 隐含寻址：操作数隐含在操作码中,可缩短指令字长;
  • 间接寻址：地址码字段是内存地址的地址,可扩大寻址范围;
  • 寄存器间接寻址：地址码字段是存内存地址的寄存器地址;
  • 相对寻址：操作数地址 + 当前PC的值;
  • 基址寻址：操作数地址 + 基址寄存器的值（一段程序中不变）;
  • 变址寻址：操作数地址 + 变址寄存器的值（随程序不断变化）;

  • 吞吐率:单位时间内流水线所完成指令或输出结果的数量;
  • 加速比:流水线的速度与等功能的非流水线的速度之比;

# 3.多级时序系统

# 1.机器周期

  • 所有指令执行过程中的一个基准时间;
  • 要考虑每条指令的步骤和每一步骤所需的时间;
  • 以最复杂的指令的执行时间或访问一次存储器的时间为基准;

# 2.时钟周期

一个机器周期可完成若干个微操作;

将一个机器周期分成若干个相等的时间段(或时钟周期);

时钟周期是控制计算机操作的最小时间单位;

# 3.多级时序系统结构

一个指令周期包含多个机器周期;

一个机器周期包含多个时钟周期;

# 4.机器速度与机器主频

在机器周期所含的时钟周期数相同的前提下,主频与平均指令执行速度成正比;

机器速度不仅与机器主频有关,还与指令周期包含的机器周期数和机器周期中包含的时钟周期数有关;

# 4.控制方式

同步控制方式:统一基准时标的时序信号控制

  • 定长的机器周期;
  • 不定长的机器周期;
  • 中央控制和局部控制相结合;

异步控制方式:无基准时标的时序信号;

联合控制方式:同步和异步结合;
