TransactionalSynchronization事务扩展

vuePress-theme-reco Heartape    2022 - 2023

TransactionalSynchronization事务扩展

Heartape 2022-11-07 springtransactional

# 1.简述

spring为我们提供l声明式事务注解@Transactional，以让我们可以简便地使用事务。 一般来说，事务都是尽量越小越好，将耗时操作放到事务中的话，会长时间占用链接，增加锁的竞争，如果数据插入修改较多的话还会导致主从同步延迟。 但有时我们在事务完成时需要做一些操作，比如发送消息队列、rpc调用其他服务等，就不能满足需求了——或者说不够优雅。

# 2.代码

/**
 * TransactionSynchronization可以优雅的实现事务扩展和异常处理
 */
public class ExampleTransactionSynchronization implements TransactionSynchronization {

    private final Runnable runnable;
    private final Runnable rollback;

    public ExampleTransactionSynchronization(Runnable runnable, Runnable rollback) {
        this.runnable = runnable;
        this.rollback = rollback;
    }

    /**
     * 可以针对不同的事务状态进行不同的操作，该方法内异常不会回滚事务（该方法内的异常不打印，afterCommit会打印，不知道是不是bug）
     * @param status STATUS_COMMITTED、STATUS_ROLLED_BACK、STATUS_UNKNOWN
     */
    @Override
    public void afterCompletion(int status) {
        // Thread[http-nio-8080-exec-1,5,main]，与上面线程一致，可见是同步执行
        System.out.println(Thread.currentThread());
        if (STATUS_COMMITTED == status) {
            runnable.run();
        } else if (STATUS_ROLLED_BACK == status) {
            rollback.run();
        } else if (STATUS_UNKNOWN == status) {
            System.out.println("STATUS_UNKNOWN");
        }
    }
}

@RestController
public class UserController {

    @Autowired
    private UserMapper userMapper;

    @GetMapping("/test")
    @Transactional
    public void test(){
        System.out.println("start");
        // Thread[http-nio-8080-exec-1,5,main]，与上面线程一致，可见是同步执行
        System.out.println(Thread.currentThread());
        userMapper.insert(new User(1, "dd"));
        if (TransactionSynchronizationManager.isActualTransactionActive()) {
            // 可以获取插入的数据
            TransactionSynchronizationManager.registerSynchronization(new ExampleTransactionSynchronization(() -> System.out.println(userMapper.selectById(1)), () -> System.out.println("执行失败")));
        }
        System.out.println("end");
    }
}
