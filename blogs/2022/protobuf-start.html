protobuf初体验

Heartape 2022-09-24 protobuf

# 1.简介

前几天做了redis私有协议RESP介绍，发现该协议虽然可读性以及处理速度不错，但是通用性太差，于是看上了protobuf。 protobuf是由谷歌开源的数据传输协议，github上用了Google's data interchange format这句话。 相比于json和其他主流协议，protobuf处理速度极快（你就说快不快吧！fastxxx:...）

# 2.使用准备

首先，需要对protobuf有一个基本的认识:

  • 二进制数据交换格式: 与json等不同，不是由明文传输。
  • 需要凭证进行序列化和反序列化: 需要编译特定的.proto文件生成相应的代码，谷歌提供了单独的编译器。

对于编译器，可以使用maven插件，较为方便，编译时Maven -> 当前模块 -> Plugins -> protobuf -> protobuf:compile

<dependencies>
    <dependency>
        <groupId>com.google.protobuf</groupId>
        <artifactId>protobuf-java</artifactId>
        <version>3.21.6</version>
    </dependency>
</dependencies>
<build>
    <extensions>
        <extension>
            <groupId>kr.motd.maven</groupId>
            <artifactId>os-maven-plugin</artifactId>
            <version>1.6.2</version>
        </extension>
    </extensions>
    <!--该插件不需要我们自己下载编译软件-->
    <plugins>
        <plugin>
            <groupId>org.xolstice.maven.plugins</groupId>
            <artifactId>protobuf-maven-plugin</artifactId>
            <version>0.6.1</version>
            <configuration>
                <protocArtifact>com.google.protobuf:protoc:3.12.0:exe:${os.detected.classifier}</protocArtifact>
                <pluginId>grpc-java</pluginId>
                <pluginArtifact>io.grpc:protoc-gen-grpc-java:1.32.1:exe:${os.detected.classifier}</pluginArtifact>
                <protoSourceRoot>src/main/resources/proto</protoSourceRoot>
                <outputDirectory>src/main/java</outputDirectory>
                <clearOutputDirectory>false</clearOutputDirectory>
            </configuration>
            <executions>
                <execution>
                    <goals>
                        <goal>compile</goal>
                        <goal>compile-custom</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

# 3.proto语法

首先贴一个较为简单的案例

// 语法版本
syntax = "proto3";
// 当前类映射包路径
package com.heartape.protobuf;
// 生成代码封装类包路径
option java_package = "com.heartape.protobuf";
// 生成代码封装类名
option java_outer_classname="ResultProto";
// 消息体
message Result {
    // 类型 字段 顺序
    int32 code = 1;
    string message = 2;
    Type type = 3;
    Data data = 4;
    // 枚举
    enum Type {
    SUCCESS = 0;
    ERROR = 1;
    }
}

message Data {
  string name = 1;
  int32 phone = 2;
  // list类型
  repeated string hobby = 3 [packed=true] ;
}

# 4.代码

public class ProtoTest {
    @Test
    public void test() {
        ResultProto.Data data = ResultProto.Data.newBuilder()
                .setName("name")
                .setPhone(1234567890)
                .addHobby("sing")
                .addHobby("basketball")
                .addHobby("game")
                .build();
        ResultProto.Result result = ResultProto.Result.newBuilder()
                .setCode(403)
                .setMessage("forbidden")
                .setData(data)
                .build();
        // 转换为二进制格式
        byte[] bytes = result.toByteArray();
        // 还原为对象
        ResultProto.Result result1 = ResultProto.Result.parseFrom(bytes);
    }
}
